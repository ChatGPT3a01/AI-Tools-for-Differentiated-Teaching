① 策略一：調整難度 — 專業語言 ➔ 口語化

把專業術語變成學生聽得懂的白話文，降低認知門檻。

📝 Prompt 範例

你是高中物理教師，請將以下內容改寫為高一生能理解的版本： 要求： 1. 專業術語用括號附上白話解釋 2. 每句不超過 20 字 3. 加上一個日常生活例子 【貼上你的簡報內容】

❌ 改寫前

「動量守恆定律指出，在一個封閉系統中，若無外力作用，則系統的總動量保持不變。」

✅ 改寫後

「動量守恆（東西的衝勁加起來不會變）：
撞球打出去，白球停了、紅球動了。
衝勁從白球「轉移」到紅球。」

② 策略二：一張 PPT ➔ 三張（拆步驟）

一張塞太多的簡報，拆成三張「一頁一重點」，降低學生的認知負荷。

📝 Prompt 範例

請將以下內容拆成 3 張簡報，每張包含： 1. 標題（10 字以內） 2. 重點（3 個要點，每點 15 字以內） 3. 講者備註（口語化，50 字以內） 【貼上你的簡報內容】
💡 為什麼要拆？
認知負荷理論：一次呈現太多資訊，學生反而什麼都記不住。
一頁一重點，學生才能逐步建立理解。

③ 策略三：融入在地情境

把課本裡的「甲地、乙地」換成學生每天走過的地方，讓學習有感連結。

📝 Prompt 範例

請改寫以下物理題目，融入在地元素： 地點用：象山步道、松山車站、信義商圈 活動用：搭捷運、體育課、園遊會 保留相同的物理概念和計算 【貼上你的題目或教材內容】

❌ 原版

「一物體以初速 v₀ 從 A 點出發，沿斜面向上運動...」

✅ 在地化

「小明從松山車站出發，騎 Ubike 上象山步道的斜坡，初速 v₀ ...」

💻 進階示範：Gemini CLI 批次處理

Gemini CLI = 在終端機裡直接跟 AI 對話，適合批量處理多份教材。

⚡ 安裝方式（一行指令）

npm install -g @anthropic-ai/gemini-cli # 或使用 npx 直接執行 npx @anthropic-ai/gemini-cli

📝 批次改寫範例

# 將整個資料夾的簡報逐一改寫 gemini "請將以下教材改寫為高一生能理解的版本" < slide01.txt # 一次處理多檔案 for f in slides/*.txt; do gemini "改寫為口語化" < "$f" > "output/${f}" done
👋 不想用 CLI？
直接用 Gemini 網頁版（gemini.google.com）也可以！
複製貼上一樣方便，只是沒辦法批次處理。

📊 三種策略一覽

策略 適用場景 效果 關鍵字
① 調整難度 學生程度差異大、基礎班看不懂原版 基礎班也聽得懂 口語化、白話文
② 拆步驟 一頁簡報塞太多內容、學生來不及吸收 一頁一重點 拆頁、分步驟
③ 在地情境 學生覺得「跟我無關」、缺乏學習動機 有感連結 換地名、換情境
💡 三種策略可以混用！
一份教材可以同時調整難度 + 拆步驟 + 融入在地情境，效果更好。
只要在 Prompt 裡把三種要求都寫進去就行。

🎯 實作練習

🎯 任務：用 Gemini 改一份教材

  1. 打開 gemini.google.com
  2. 從你的科目中複製一段教材（課本、講義、簡報都行）
  3. 選擇一種策略，貼上對應的 Prompt
  4. 檢視 AI 的改寫結果，視需要微調
  5. 試試看三種策略各做一次，比較差異
📋 小提醒
  • 可以把三種策略寫在同一個 Prompt
  • Gemini 的結果不一定完美，人工微調是正常的
  • 不同科目的效果可能不同，多試幾次找到最佳寫法
