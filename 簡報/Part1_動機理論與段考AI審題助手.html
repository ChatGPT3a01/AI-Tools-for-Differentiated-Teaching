經典研究

內部動機 vs 外部動機：畫畫實驗

🎨 A 組（無獎勵）

自由畫畫
↓
隔天繼續畫
↓
✅ 動機不變

🍬 B 組（有糖果）

知道畫完有糖果
↓
隔天不想畫了
↓
❌ 動機下降
⚠️ 結論：外部獎勵會削弱內部動機
這就是為什麼「加分」不是萬靈丹。
當學生為了獎勵才做，拿掉獎勵就不做了。
理論延伸

動機的四個象限

🌟 內部正向

「我想做！」

例：因為喜歡畫畫而畫

最持久

😬 內部負向

「我不做會不安」

例：不寫作業會良心不安

次持久

🎁 外部正向

「做了有獎勵」

例：加分、糖果、獎狀

短暫

😠 外部負向

「不做會被處罰」

例：不寫要罰站、扣分

最短暫
動機持久度： 內部正向 > 內部負向 > 外部正向 > 外部負向
🎯 自主性策略

選擇增加動機：滷肉飯 + 炸豆腐策略

🍲 故事：
一家店只賣滷肉飯，Google 評分 4.0。
加上一道不太好吃的炸豆腐後，客人可以「選擇」
→ 大部分客人選了滷肉飯 → 滿意度反而提高了！

📚 教學上的應用

原本 只有一種作業形式 → 寫兩頁報告
改為 加上一個「更難」的選項 → 寫報告 + 錄口頭說明
效果 學生「選擇」只寫報告 → 覺得輕鬆 → 動機提升
💡 你不需要真的降低難度，
只需要讓學生覺得「這是我自己選的」

🛠️ 實作步驟

Step 1：建立 Google Sheets

  1. 開啟 Google Sheets（sheets.google.com）
  2. 建立新試算表，命名為「段考AI審題助手」
  3. 在 A 欄貼上段考題目（含題號和選項）
    每題一列，或全部貼在 A1 也可以
🛠️ 實作步驟

Step 2：開啟 Apps Script 編輯器

  1. 在 Google Sheets 上方選單，點「擴充功能」→「Apps Script」
  2. 會開啟新的 Apps Script 編輯器視窗
  3. 刪除預設的 myFunction 程式碼（全選刪除即可）
🛠️ 實作步驟

Step 3：設定 Gemini API 金鑰

  1. 前往 Google AI Studio（aistudio.google.com）
  2. 點「Get API Key」→ 建立 API 金鑰 → 複製
  3. 回到 Apps Script → 左邊齒輪「專案設定」→「指令碼屬性」
  4. 新增屬性：
    名稱：GEMINI_API_KEY
    值：你的 API 金鑰
⚠️ 安全提醒：API 金鑰等同密碼，不要分享給他人！
放在「指令碼屬性」中，程式碼不會直接暴露金鑰。
🛠️ 實作步驟

Step 4：貼上程式碼

  1. 打開程式碼檔案：工具1_段考AI審題助手.js
  2. 全選（Ctrl+A）→ 複製（Ctrl+C）
  3. 回到 Apps Script 編輯器 → 全選刪除舊程式碼 → 貼上（Ctrl+V）
  4. 按 Ctrl+S 儲存
🛠️ 實作步驟

Step 5：儲存並授權

  1. 上方工具列，函式下拉選單選 onOpen → 點「▶ 執行」
  2. 會跳出 Google 授權視窗
  3. 選你的 Google 帳號 → 點「進階」→「前往...（不安全）」→「允許」
💡 說明：首次執行需要授權 Google 存取你的 Sheets。
這是正常流程，授權一次之後就不會再跳出了。
🛠️ 實作步驟

Step 6：使用工具

  1. 回到 Google Sheets → 按 F5 重新整理頁面
  2. 上方會出現自訂選單「🤖 AI 審題助手」
  3. 點「📊 分析段考試題」
  4. 等待約 30 秒 ~ 1 分鐘，AI 分析完成！
🎉 完成！分析結果會自動產生在三個新的工作表中。

✍️ Prompt 技巧

Prompt 設計要點

好的 Prompt = 好的 AI 輸出。掌握這四個關鍵：

🎭

角色設定

「你是一位嚴謹的高中物理教師」
→ 讓 AI 用專業視角分析試題

📋

結構化輸出

要求回傳 JSON 格式
→ 程式可自動解析並填入表格

✅

明確項目

列出需要分析的 4 大項目
→ 避免 AI 遺漏或答非所問

🌡️

低 Temperature

設為 0.2（穩定模式）
→ 每次分析結果一致、可重複

💬 記住：Prompt 越具體，AI 輸出越精準。
不要說「分析這份考卷」，要說「請從以下 4 個面向分析...」

📋 提示詞模板：打造你自己的段考 AI 審題助手

複製以下提示詞，貼到 Gemini / ChatGPT / Claude，即可取得完整的 Google Apps Script 程式碼：

請幫我寫一個 Google Apps Script（GAS），功能是「段考 AI 審題助手」。

需求：
1. 在 Google Sheets 中建立自訂選單「🤖 AI 審題助手」
2. 使用者在 Sheet1 的 A 欄貼上段考題目（含題號和選項）
3. 點選單後，程式讀取 A 欄內容，呼叫 Gemini API（模型：gemini-2.5-flash）進行分析
4. API Key 存在「指令碼屬性」中，名稱為 GEMINI_API_KEY

分析內容（要求 AI 以 JSON 格式回傳）：
- 雙向細目表：每題對應的「單元」和「認知層次」（記憶/理解/應用/分析/評鑑/創造）
- 難易度分析：每題的難度（簡單/中等/困難）+ 統計百分比
- 檢核與建議：未覆蓋的單元、出題偏重、難度均衡性、改進建議

輸出：自動建立三個工作表（雙向細目表、難易度分析、檢核與建議），並格式化表格

技術要求：
- temperature 設為 0.2（穩定分析）
- 使用 UrlFetchApp.fetch 呼叫 Gemini API
- 包含錯誤處理和使用者提示對話框
- Prompt 中設定角色為「嚴謹的高中教師」
💡 提示：把上面的提示詞貼到 AI 後，你會得到完整的 .gs 程式碼。
你也可以根據自己的科目修改 Prompt 中的角色設定和分析項目。
